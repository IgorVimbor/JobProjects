------------------- Запуск Nginx на Windows: правильные команды ---------------------

Обе команды рабочие, но они имеют разное поведение и предназначены для разных сценариев:

1 .\nginx.exe (без start)
Запускает Nginx в текущем окне командной строки

Окно командной строки остается занятым процессом Nginx

Если закрыть окно - Nginx остановится

Подходит для: тестирования, отладки, когда нужно видеть вывод в реальном времени

2 start nginx.exe
Запускает Nginx в фоновом режиме как отдельный процесс

Командная строка сразу освобождается для других команд

Nginx продолжает работать после закрытия окна командной строки

Подходит для: production-режима, постоянной работы сервера

Рекомендация для вашего случая
Для работы сайта в локальной сети используйте start nginx - это правильный выбор для production-окружения.

-----------------------------------------------------------------------

Полный процесс запуска:
cmd
# Перейдите в папку nginx
cd C:\nginx

# Остановите предыдущую версию (если была запущена)
.\nginx.exe -s quit

# Запустите Nginx в фоновом режиме (правильно для production)
start nginx.exe

------------------------------------------------------------------------

Проверка работы:
cmd
# Проверить, что Nginx запущен
tasklist /fi "imagename eq nginx.exe"

-------------------------------------------------------------------------

Управление Nginx:
cmd
# Graceful shutdown (корректная остановка)
nginx -s quit

# Быстрая остановка
nginx -s stop

# Перезагрузка конфигурации без остановки сервера
nginx -s reload

# Повторное открытие log-файлов
nginx -s reopen

---------------------------------------------------------------------------

Важные замечания:
1. При использовании start nginx процесс продолжает работать после закрытия командной строки.

2. Если вы запускаете через nginx.exe (без start), то для постоянной работы нельзя закрывать окно командной строки.

3. Для production-режима на Windows лучше всего настроить Nginx как службу Windows, чтобы он автоматически запускался при загрузке системы.

4. Если возникают проблемы с правами доступа, запускайте командную строку от имени администратора.
