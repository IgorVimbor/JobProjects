------------ Запуск проекта (если дефолтный settings.py) ---------------
python manage.py runserver

------------ Запуск проекта в режиме разработки (settings/development.py) -----------
python manage.py runserver --settings=reclamationhub.settings.development

------------ Запуск проекта в продакшене (settings/production.py) ---------------

python manage.py runserver 0.0.0.0:8000 --settings=reclamationhub.settings.production

-- При таком подключении будет использоваться сервер Django, который в режиме production
-- не подхватывает медиа файлы из папки media

------------ Как подключиться другим устройствам
На моем рабочем ПК: http://localhost:8000

На других ПК в сети: http://192.168.0.191:8000 (http://ваш_IP:8000)

-- Порядок ручного запуска:

1. Сначала запустите Django-сервер (Waitress):
# Откройте командную строку или PowerShell

-- Перейдите в папку вашего проекта и активируйте виртуальное окружение
bash
cd C:\путь\к\вашему\project
venv\Scripts\activate

-- Запустите Waitress
# waitress-serve --port=8000 myproject.wsgi:application

waitress-serve --port=8000 reclamationhub.wsgi:application

2. Затем в отдельном окне запустите Nginx:
# Откройте новую командную строку или PowerShell
bash
cd C:\nginx-1.28.0
.\nginx.exe (или start nginx.exe)

-------------- Важные моменты -----------------
Порядок важен: Сначала Django, потом Nginx

Два окна терминала: Waitress будет работать в первом окне, а Nginx во втором

Проверка работы: После запуска обоих серверов откройте браузер и перейдите по адресу http://localhost

-------------- Для остановки серверов --------------
1. Остановите Nginx:
bash
.\nginx -s stop

2. Остановите Waitress: Нажмите Ctrl+C в окне, где работает Waitress

Пока вы не настроили автоматический запуск (пункт 6), вам нужно будет вручную запускать оба сервера:

Запуск сервера приложений (Django через Waitress):

Откройте командную строку (или PowerShell) в вашем проекте.

Активируйте виртуальное окружение (если используете).

Выполните команду: waitress-serve --port=8000 myproject.wsgi:application

Запуск Nginx:

Откройте другую командную строку (так как первая будет занята Waitress).

Перейдите в папку Nginx (например, cd C:\nginx).

Запустите Nginx: start nginx

Важно: Waitress должен быть запущен до того, как Nginx начнет перенаправлять на него запросы. Обычно порядок запуска не критичен, потому что Nginx будет пытаться подключиться к Waitress при каждом запросе, но если Waitress не запущен, то при обращении к сайту вы получите ошибку 502 Bad Gateway.

Также обратите внимание, что после внесения изменений в конфигурацию Nginx или в код Django, вам может потребоваться перезапуск одного или обоих серверов.
