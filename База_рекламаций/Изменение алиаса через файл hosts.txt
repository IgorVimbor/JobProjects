--------------------- Создание алиаса через файл hosts ---------------------------

------------ Шаг 1: Редактирование файл hosts
-- Откройте как администратор:

C:\Windows\System32\drivers\etc\hosts

-- Добавьте в конец файла:

127.0.0.1    bza-otk
192.168.0.191    bza-otk

------------ Шаг 2: Настройте Nginx
-- В конфигурации C:\nginx\conf\nginx.conf измените:

server {
    listen 80;
    server_name localhost 127.0.0.1 192.168.0.191 bza-otk;
    # ... остальная конфигурация
}

------------- Шаг 3: Обновите Django настройки
-- В production.py добавьте:

ALLOWED_HOSTS = [
    '192.168.0.191',
    'localhost',
    '127.0.0.1',
    'bza-otk',       # ← новый алиас
]

CSRF_TRUSTED_ORIGINS = [
    'http://192.168.0.191',
    'http://localhost',
    'http://127.0.0.1',
    'http://bza-otk',   # ← новый алиас
]

-------------- Шаг 4: Перезапустите Nginx
.\nginx.exe -s reload

Теперь получен доступ по красивому адресу:
✅ http://bza-otk/admin/


-------------------------------- Где редактировать hosts -----------------------------
✅ На рабочем ПК (где сервер):
C:\Windows\System32\drivers\etc\hosts
-- Добавить:
127.0.0.1    bza-otk
192.168.0.191    bza-otk

✅ На домашнем ПК (если хотите тестировать):
C:\Windows\System32\drivers\etc\hosts
-- Добавить:
192.168.0.191    bza-otk

✅ На компьютерах коллег (все 5 пользователей):
C:\Windows\System32\drivers\etc\hosts
-- Добавить:
192.168.0.191    bza-otk


-- Принцип работы:
Файл hosts говорит конкретному компьютеру: "когда видишь адрес bza-otk, иди на IP 192.168.0.191".

-- Альтернативный вариант (если есть корпоративный DNS):
Обратитесь к системному администратору - он может добавить запись в корпоративный DNS. Тогда hosts редактировать не нужно будет.

-- На сервере (рабочий ПК):
Nginx и Django уже настроены правильно, дополнительно ничего не нужно!
